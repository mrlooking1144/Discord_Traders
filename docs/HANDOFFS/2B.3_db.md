# Milestone

2B.3 — database/db.py

# Summary

Implemented the SQLite connection and initialization layer for the Version 1
database, using only the Python standard library. `database/db.py` exposes
two public functions:

- `get_connection(config: DatabaseConfig) -> sqlite3.Connection` — opens a
  SQLite connection against `config.db_path`, applies
  `config.busy_timeout_seconds` as the connection's busy timeout, enables
  foreign-key enforcement (`PRAGMA foreign_keys = ON`), and sets
  `connection.row_factory = sqlite3.Row`.
- `initialize_database(config: DatabaseConfig) -> None` — opens its own
  connection via `get_connection`, reads `database/schema.sql` (located
  relative to the module file, not the process working directory), and
  applies it with `executescript`. The existing schema's exclusive use of
  `IF NOT EXISTS` makes this safe to call repeatedly. The connection is
  always closed in a `finally` block, including if schema application fails.

No business logic, repository queries, transaction workflows, application
imports, Discord logic, parser logic, or UI logic was added. No schema was
duplicated or altered — `schema.sql` is read and executed as-is.
