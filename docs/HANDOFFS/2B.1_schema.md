# Milestone

2B.1 — database/schema.sql

# Summary

Implemented the Version 1 SQLite schema exactly as specified in
`docs/DATABASE_DESIGN_V1.md`: the `sources`, `traders`, `raw_messages`,
`trade_signals`, and `trade_signal_edits` tables, with all partial-unique
constraints, non-unique indexes, and foreign keys as designed. No other
modules, tests, or documentation were touched.

# Files Modified

- `database/schema.sql` (new)

# Tests Executed

No test file was created for this milestone (`tests/` remains empty). The
schema was validated by applying it to a temporary, empty SQLite database via
Python's built-in `sqlite3` module:

- Applied `database/schema.sql` to a fresh temp-file database.
- Re-applied it a second time to confirm idempotency (`IF NOT EXISTS` guards
  on all `CREATE TABLE` / `CREATE INDEX` statements).
- Queried `sqlite_master` to confirm all 5 tables and all expected indexes
  (including the two partial-unique indexes) were created correctly.
- Queried `PRAGMA foreign_key_list(<table>)` for every table to confirm all 5
  foreign keys were created correctly.

All checks passed with no errors.

# Known Limitations

- `PRAGMA foreign_keys = ON` is not set anywhere yet — SQLite does not enforce
  foreign keys by default, and this pragma is a per-connection setting rather
  than persisted schema state. It is expected to be set in `database/db.py`
  (Milestone 2B.3), not in `schema.sql`.
- No automated test file exists yet asserting schema structure; validation for
  this milestone was manual (see Tests Executed above).
- No application code (config, connection layer, models, repository, service)
  exists yet — the schema is not yet usable by the application.
