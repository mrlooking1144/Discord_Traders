# Milestone

2C.2 — Manual Message Entry UI (`app/app.py`)

# Objective

Add a manual message entry flow to the Streamlit UI allowing the user to
paste raw trade message text, invoke the Milestone 2C.1 parser
(`app.parser.parse_message`), and review the resulting structured trade data
before it is persisted. Per `docs/IMPLEMENTATION_ROADMAP.md`, no persistence
occurs in this milestone.

# Files Implemented

- `app/app.py` — manual raw-message text entry UI: a text area, a "Parse
  Message" button that invokes `parse_message()`, and display of either the
  structured signal(s) (`st.json`) or a "No trade signals found" warning.
- `tests/test_app.py` (new) — UI/smoke tests using Streamlit's
  `streamlit.testing.v1.AppTest` harness.
- `requirements.txt` (new) — pins `streamlit==1.59.2`, the exact version
  installed and tested during this milestone.

# What Was Implemented

- A `st.text_area` for pasting raw trade message text.
- A `st.button` ("Parse Message") that, on click, calls
  `app.parser.parse_message()` on the pasted text.
- If one or more signals are found: a success message with the count,
  followed by each structured signal dict rendered via `st.json`.
- If no signals are found (including empty/unparseable input): a
  "No trade signals found in this message." warning.
- No calls to `TradeService`, `repository.py`, or the database anywhere in
  this file.

# Test Results

**196/196 tests passing** (191 carried over from 2B.1–2C.1 + 5 new in
`tests/test_app.py`) via `python -m unittest discover -s tests`.

Coverage includes: successful single-signal parse and display, multi-signal
parse and display, no-signal ("nothing found") display for unparseable text,
no-signal display for empty input, and no result shown prior to the button
being clicked.

# Known Limitations

- No persistence: parsed signals are displayed only and are discarded on
  the next rerun. Wiring to `TradeService.ingest_message()` is explicitly
  deferred to Milestone 2C.3.
- No source/trader/message metadata is collected in this milestone (source
  name, external message ID, etc.) — that is part of the 2C.3 integration.
- Streamlit is now a required dependency (`requirements.txt`); no other
  application code changed as a result of adding it.

# Next Milestone
