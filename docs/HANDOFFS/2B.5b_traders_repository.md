# Milestone

2B.5b ‚Äî Traders Repository (`database/repository.py`)

# Summary

Extended `database/repository.py` (built up incrementally per Milestone 2B.5)
with trader-related data-access functions, the second slice after 2B.5a
(sources). As with the sources functions, all trader functions accept an
existing `sqlite3.Connection` supplied by the caller; the module never opens
a connection itself and never commits or rolls back, leaving transaction
control entirely to the caller. Rows are mapped to the existing `Trader`
dataclass from `database/models.py` via a new private helper,
`_row_to_trader()`, which performs mapping only ‚Äî no SQL or business logic.
All SQL is parameterized.

Trader name validation (rejecting empty and whitespace-only names) follows
the same pattern established in 2B.5a: a private `_validate_trader_name()`
helper raising the built-in `ValueError`, no new exception type introduced.
Uniqueness of `(source_id, external_trader_id)` and the `source_id` foreign
key are left entirely to SQLite's schema-level enforcement ‚Äî `create_trader`
does not catch or translate `sqlite3.IntegrityError`, so callers see the
database's own constraint violations directly.

This slice covers only `traders` table access, per the roadmap's incremental
plan for Milestone 2B.5 (2B.5a‚Äì2B.5e).

# Repository Functions Added

- `create_trader(conn, source_id, name, external_trader_id=None) -> Trader` ‚Äî
  inserts one trader row and returns the complete persisted `Trader`,
  including the generated `id` and DB-defaulted `created_at` (re-read via a
  follow-up `SELECT` on `lastrowid`, since SQLite does not return
  DB-default column values from an `INSERT`); raises `ValueError` for an
  empty or whitespace-only name; raises `sqlite3.IntegrityError` uncaught for
  a missing `source_id` or a duplicate non-null `external_trader_id` within
  the same source.
- `get_trader_by_external_id(conn, source_id, external_trader_id) -> Trader | None` ‚Äî
  exact lookup scoped to both `source_id` and `external_trader_id`; returns
  `None` if no row matches. Never performs a global, cross-source lookup.
- `get_traders_by_name(conn, source_id, name) -> list[Trader]` ‚Äî exact-name
  lookup scoped to `source_id`, returning every match ordered by `id`;
  duplicate trader names within a source are valid data and are all
  returned; returns an empty list when no rows match.

# Validation Rules

- Trader `name` must not be empty or whitespace-only (`_validate_trader_name`,
  reused only by `create_trader`; the get-functions are pure lookups and
  perform no validation, consistent with the read/write split from 2B.5a).
- `external_trader_id` may be `None`; multiple traders with `NULL`
  `external_trader_id` under the same source are permitted (SQLite's partial
  unique index only applies `WHERE external_trader_id IS NOT NULL`, per
  `docs/DATABASE_DESIGN_V1.md` Section 3).
- No new exception class was introduced; validation continues to use the
  built-in `ValueError`, matching the resolution agreed for 2B.5a since
  `database/models.py` defines no exception types to reuse.

# Transaction Rules

- No function in this module opens or closes a connection, or calls
  `commit()` / `rollback()`. The caller is solely responsible for
  transaction boundaries.
- `sqlite3.IntegrityError` (foreign key violation, duplicate
  `(source_id, external_trader_id)`) propagates uncaught from `create_trader`
  ‚Äî the module does not swallow or reinterpret database-enforced
  constraints.

# Testing Summary

`tests/test_repository_sources.py` gained a new `TradersRepositoryTests`
class (16 tests) alongside the unchanged, still-passing `SourcesRepositoryTests`
class from 2B.5a (11 tests), using the standard library's `unittest` against
a temporary file-based SQLite database:

- Creating a trader with and without an `external_trader_id`.
- Confirming the returned `Trader` has a generated `id` and `created_at`.
- External-ID lookup: correct match, scoped to `source_id`, and `None` when
  missing.
- Name lookup: correct matches, duplicate names within a source all
  returned, scoped to `source_id`, and empty list when missing.
- Duplicate non-null `external_trader_id` within the same source raises
  `sqlite3.IntegrityError`.
- The same `external_trader_id` is allowed across two different sources.
- Multiple traders with `external_trader_id=None` are allowed under the same
  source.
- Empty and whitespace-only names raise `ValueError`.
- Repository writes are not committed automatically (verified with two
  connections to the same temporary database file).

**Result: 27/27 tests passing** (11 sources + 16 traders) via
`python -m unittest discover -s tests -v`.
